## YAML 头部（必需）

文件开头必须包含 YAML 头部，用 `---` 包裹：

```yaml
---
emoji: "🚀"           # 封面 Emoji，1 个字符
title: "大标题"        # 封面标题，建议≤15 字
subtitle: "副标题"     # 封面副标题，建议≤15 字
---
```

### 字段说明

| 字段 | 类型 | 必需 | 说明 | 示例 |
|------|------|------|------|------|
| `emoji` | string | ✅ | 封面装饰 Emoji | `"🚀"` |
| `title` | string | ✅ | 封面大标题 | `"5 个效率神器"` |
| `subtitle` | string | ✅ | 封面副标题 | `"打工人必备"` |

## 正文格式

### 标题层级

```markdown
# 一级标题 - 卡片大标题

## 二级标题 - 卡片小标题

### 三级标题 - 较少使用
```

### 段落

```markdown
这是普通段落，直接书写即可。

段落之间用空行分隔。
```

### 强调文本

```markdown
**加粗文本** - 重要内容

*斜体文本* - 强调内容

***加粗斜体*** - 强烈强调
```

### 列表

```markdown
## 无序列表

- 项目一
- 项目二
  - 子项目
  - 子项目

## 有序列表

1. 第一步
2. 第二步
3. 第三步
```

### 引用

```markdown
> 这是一段引用内容
> 通常用于强调或引用他人的话

> 多段引用
> 可以连续多行
```

### 代码

```markdown
## 行内代码

使用 `code` 标记行内代码

## 代码块

```python
def hello():
    print("Hello, World!")
```

## 带语言标识的代码块

```javascript
console.log("Hello");
```
```

## 分页控制

### 手动分页

使用 `---` 在内容中创建分页：

```markdown
# 第一页内容

...

---

# 第二页内容

...
```

### 自动分页

使用 `-m auto-split` 参数让脚本自动切分长内容。

---

## 最佳实践

### ✅ 推荐

- 标题简洁有力（≤15 字）
- 段落短小精悍（3-5 行）
- 适当使用 Emoji 增加可读性
- 用列表代替长段落
- 重要内容用**加粗**标记

### ❌ 避免

- 过长的段落（超过 10 行）
- 复杂的嵌套列表
- 过多层级的标题
- 大段代码（超过 20 行）
- 外部图片链接（不支持）

---

## 字数建议

| 卡片类型 | 建议字数 | 最大字数 |
|----------|----------|----------|
| 封面标题 | ≤15 字 | 20 字 |
| 封面副标题 | ≤15 字 | 20 字 |
| 单页正文 | 100-200 字 | 300 字 |
| 多页总内容 | 300-800 字 | 1000 字 |

## 完整模板

复制以下模板开始创作：

```markdown
---
emoji: "🚀"
title: "你的标题"
subtitle: "你的副标题"
---

# 第一部分

> 核心观点或引言

## 要点一

- 内容一
- 内容二
- 内容三

## 要点二

**重要提示**：这里是详细说明。

---

# 第二部分

继续更多内容...

---

# 总结

1. 要点回顾
2. 行动建议
3. 额外资源
```

## 常见问题

### Q: Emoji 不显示怎么办？

A: 确保使用标准 Unicode Emoji，不要用文字描述。

### Q: 如何控制每页内容量？

A: 使用 `---` 手动分页，或用 `-m auto-split` 自动切分。

### Q: 可以插入图片吗？

A: 当前版本不支持外部图片，纯文字 +Emoji 渲染。
